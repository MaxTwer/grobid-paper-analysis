Grobid Text Analysis of Open-Access Articles

Project Description

This project performs a basic text analysis over open access scientific articles using Grobid to extract structured information from PDF files.

The objective is to automatically process the papers and generate:

 A keyword cloud based on the abstracts of the articles.
 A visualization of the number of figures per article.
 A list of the links found in each paper.

The analysis is performed using Python scripts over the XML files generated by Grobid.



Dataset

The dataset used consists of 10 open-access research papers obtained from arXiv.

The papers are stored in:


pdf/


Each PDF is processed using Grobid to generate a structured XML file, which is stored in:


tei/

 Methodology

 1. PDF Processing with Grobid

The script `proceso.sh` processes the PDF files using Grobid and generates the XML files.


These XML files are then used for the analysis.



 2. Keyword Cloud from Abstracts

The script keyword_cloud.py performs the following steps:

1. Extracts the `<abstract>` section from each XML file.
2. Cleans the text (removing stopwords and punctuation).
3. Computes word frequencies.
4. Generates a word cloud visualization.

Output: keyword_cloud.png



3. Figures per Article Visualization

The script: figures_per_article.py counts the number of <figure> tags in each TEI XML file.

The result is visualized using a bar chart, where each bar corresponds to a paper.

Output:

figures_per_article.png




4. Extracting Links from Articles

The script: links_per_article.py extracts links from the XML files using the `<ref>` and `<ptr>` elements generated by Grobid.

The script filters out default links generated by Grobid 

Output:

links_per_article.txt




 Validation of Results

 Keyword Cloud Validation

To validate the keyword cloud:

 The extracted abstracts were manually compared with the original PDF abstracts.
 Stopwords were removed to ensure meaningful keywords.
 The most frequent terms were manually inspected to confirm they match the article topics.



 Figures per Article Validation

Validation was performed by:

 Manually counting figures in a subset of the PDF papers.
 Comparing the count with the number of <figure> tags in the TEI XML files.

The counts were consistent across the tested samples.


 Links Extraction Validation

Validation steps:

 The extracted links were manually compared with links visible in the PDF.
 Default Grobid links were filtered from the results.
 Multiple papers were manually inspected to ensure correct extraction.



 Reproducibility

To reproduce the analysis:

1. Install Grobid
2. Place the PDF files inside the `pdf/` folder
3. Start grobid the background:

docker run -d --rm --init \-p 8070:8070 \-e JAVA_OPTS="-Xmx4g -Xms2g" \--name grobid \grobid/grobid:0.8.0

3. Run the pipeline script:


./run_analysis.sh


This script will:

1. Clean previous files in tei
2. Generate new XML files using Grobid
3. Run the three analysis scripts
